基于 ADHD 友好设计的低阻力学习规划 Web 应用研究
——以 FocusFlow Study Planner 为例

作者：________

摘  要：针对大学生在课程学习、项目推进与自主复习过程中普遍存在的任务启动困难、长任务列表带来的认知负担以及专注与休息节奏难以维持等问题，本文设计并实现了一款基于 ADHD 友好设计理念的低阻力学习规划 Web 应用 FocusFlow Study Planner。该系统以“减少视觉噪音、降低输入门槛、突出最小下一步、强化专注闭环”为核心设计原则，围绕任务快速录入、今日聚焦、番茄钟式全屏倒计时、休息模式、中英双语切换与轻量设置等功能展开实现。系统采用 React 前端框架与 LocalStorage 本地持久化机制，构建了面向学生用户的单页 Web 应用。研究结果表明，该项目能够将传统待办清单的“记录任务”逻辑扩展为“支持启动与持续行动”的交互流程，更契合注意力易分散用户在真实学习场景中的使用需求。本文最后总结了系统的应用价值、设计特点及后续可扩展方向。

关键词：ADHD 友好设计；时间管理；番茄工作法；学习规划；Web 应用

引言

随着高校课程任务复杂度不断提升，学生在学习过程中不仅需要处理作业、论文、项目、展示等多类型任务，还需要在有限时间内完成优先级判断、注意力切换与持续专注。传统待办类工具虽然能够帮助用户记录事项，但在“如何启动任务”“如何维持专注”“如何在高压力状态下减少界面负担”等方面支持有限。已有研究指出，注意缺陷多动障碍（ADHD）及其相关执行功能困难会显著影响个体的计划、组织、工作记忆与任务完成效率[1][2]。与此同时，大学阶段学生的执行任务能力与认知灵活性又与学业表现呈现显著相关关系[3]。因此，围绕“降低任务启动阻力”和“减少认知过载”来设计学习规划工具，具有明确的现实意义。基于此，本文选取 FocusFlow Study Planner 作为研究对象，围绕其设计背景、系统功能、交互实现及应用价值进行总结分析。

# 1 项目背景与需求分析

## 1.1 问题定义

本项目要解决的问题，并不是简单的“学生缺少一个待办清单”，而是学生在面对学习任务时常常会遭遇以下几类更本质的困难：其一，大型任务边界模糊，用户知道“要写论文”或“要完成项目”，但不知道第一步从哪里开始；其二，任务量一旦增多，常规列表界面容易形成视觉压迫，用户在浏览任务时已经消耗了大量认知资源；其三，计划工具与专注工具常常分离，用户即使写下了任务，也未必能顺畅进入执行状态；其四，一轮专注结束后，如果缺少明确的休息与重启节奏，用户往往难以继续维持稳定的学习流程。上述问题在注意力调节能力较弱、执行功能负担较重的学生群体中表现得尤为明显[1][2]。

## 1.2 目标用户与应用场景

FocusFlow 的目标用户主要为高校学生与研究生，尤其适用于存在任务启动困难、学习拖延、界面过载敏感、注意力易分散等问题的用户。典型使用场景包括：课程作业截止日前的分段推进、毕业设计或项目开发中的小步执行、论文写作中的引言或文献综述拆解、日常复习任务的轻量管理等。与通用型任务工具相比，本项目更强调“在压力情境中仍然容易开始”，而不是追求大而全的项目管理能力。

## 1.3 项目目标

本项目的设计目标包括三个层面。第一，在产品层面，构建一个真正可运行的 Web 应用，而非静态原型；第二，在交互层面，通过最小下一步、今日聚焦与大号倒计时等机制，降低用户从“记录任务”到“开始行动”的心理门槛；第三，在课程要求层面，围绕信息技术应用开发这一主题，展示需求分析、界面设计、状态管理、持久化存储与用户场景导向设计等综合能力。

# 2 系统设计与实现

## 2.1 功能设计

FocusFlow 的核心功能围绕学习任务的完整闭环展开。系统首先提供低阻力快速录入表单，用户仅需填写任务标题、课程或项目、最小下一步、截止日期及精力级别，即可快速创建任务。这里的“最小下一步”设计是项目的关键功能之一，它将抽象任务拆解为可立即执行的微动作，例如把“完成文献综述”转化为“打开文章并写一句摘要”，以减少任务启动阻力。

在任务展示方面，系统提供“今日聚焦”模块，仅呈现少量优先任务，而不是将全部事项同时暴露给用户。该设计遵循 ADHD 友好界面的基本原则，即通过减少同时出现的信息量，降低视觉噪音与选择负担。除此之外，系统还保留任务库、状态筛选、课程筛选和截止日期展示，用于支持用户对长期任务进行管理。

在执行阶段，用户点击“现在开始”后，页面会进入全屏专注模式，界面主体转变为番茄钟式大号倒计时，显示当前任务、下一步动作和专注状态。该设计借鉴了 Pomodoro Technique 对时间切片与短时专注的处理方法[4]，使“计划”与“执行”不再割裂。为了形成更完整的时间管理闭环，系统还在专注结束后引入休息模式，支持自动进入休息轮次，并允许用户在设置面板中调整专注时长、深度专注时长、休息时长以及是否自动进入休息。

## 2.2 交互与界面设计

本项目的界面设计重点不在于堆叠功能，而在于减少不必要的解释性文字与信息层级。实践过程中，系统曾一度保留较多产品说明性文案，但从真实工具使用角度看，用户并不需要在每次打开应用时重新阅读这些说明。为此，后续设计中删除了首页大段介绍文字、理念卡片和冗余注释，保留品牌标识、核心操作入口与必要状态信息，使页面更接近真正的生产力工具。

在视觉风格上，系统采用柔和背景、低饱和色彩和较大的留白，并支持中英文双语切换与昼夜主题切换。计时界面进一步强化“单任务优先”的设计原则，用全屏大倒计时作为主视觉，弱化周边干扰信息，使用户在专注阶段只关注当前任务和剩余时间。这种设计方式更符合注意力脆弱用户对“少而明确”的界面需求。

## 2.3 技术实现

从技术角度看，FocusFlow 基于 React 构建，采用函数式组件与 Hooks 完成状态管理。任务数据、语言偏好和专注设置均通过浏览器 LocalStorage 进行本地持久化，以降低部署复杂度并保证前端原型的可运行性。系统内部实现了任务模型归一化、筛选排序、专注倒计时、休息倒计时、主题切换和语言切换等逻辑，同时通过简单测试验证了“双语切换”和“进入专注模式”两项关键交互行为。整体上，该项目展示了前端单页应用开发中常见的状态管理、交互编排、界面响应式布局和本地持久化实现方法。

# 3 项目价值与特点分析

## 3.1 对用户的实际价值

相较于传统待办工具，FocusFlow 的价值不在于记录更多任务，而在于帮助用户更容易开始任务。对于存在启动困难的学生而言，“知道该做什么”与“真的能开始做”之间往往存在明显断层。本项目通过最小下一步、今日聚焦、大号倒计时和休息模式，将这一断层进一步缩短，使用户更容易从规划状态进入行动状态。此外，系统的轻量设置能力也使其更适合个体差异显著的学习者，不同用户可以根据自己的注意力节奏调整专注轮次与休息时长。

## 3.2 与一般待办应用的区别

一般待办应用更关注记录、分类和提醒，而 FocusFlow 更强调执行功能支持。前者更像事务清单，后者更像面向学习者的行动启动工具。其差异主要体现在三个方面：一是强调任务拆解而非任务堆叠；二是强调聚焦与倒计时闭环，而非停留在列表浏览；三是强调低认知负担的界面，而非展示大量统计与复杂管理结构。正因为如此，本项目不只是一个“做了皮肤美化的待办应用”，而是围绕特定用户问题进行了定向设计的 Web 产品原型。

## 3.3 对课程要求的适配性

从课程作业角度看，本项目较好地满足了“围绕现实问题开发 Web 应用”的要求。它具有明确的目标用户、现实的教育场景、可运行的功能系统以及可说明的技术路径。同时，该项目也具备进一步扩展为更完整系统的空间，例如后续可加入账户系统、云端同步、学习数据分析与跨设备访问等模块。因此，无论从问题导向、应用开发，还是从课程展示效果来看，FocusFlow 都具备较好的适配性。

# 4 结语

本文围绕 FocusFlow Study Planner 的项目背景、需求分析、功能设计、交互特点与实现方式进行了总结。项目以 ADHD 友好设计为切入点，将学习任务管理、番茄钟式专注执行与休息闭环整合到同一 Web 应用中，尝试解决学生群体在真实学习场景中的任务启动与持续专注问题。实践表明，该项目不仅具有明确的用户价值，也能够体现 Web 应用开发中对需求分析、界面设计与交互逻辑实现的综合理解。未来，系统仍可在云同步、数据分析、用户研究与个性化推荐等方面进一步完善，从而从课程项目原型逐步发展为更成熟的学习支持工具。
