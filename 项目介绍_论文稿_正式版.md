基于 ADHD 友好设计的低阻力学习规划 Web 应用研究
——以 FocusFlow Study Planner 为例

作者：________

摘  要：在高校学习场景中，学生普遍需要同时处理课程作业、项目开发、论文写作与阶段性复习等多类型任务。然而，传统待办工具在任务启动支持、注意力维持与认知负担控制方面存在明显不足，尤其难以满足存在执行功能困难或注意力易分散用户的实际需求。为解决上述问题，本文设计并实现了一款面向学生群体的 ADHD 友好型低阻力学习规划 Web 应用 FocusFlow Study Planner。该系统以“降低任务启动门槛、减少视觉噪音、强化最小下一步、构建专注与休息闭环”为核心思路，围绕任务快速录入、今日聚焦、番茄钟式全屏倒计时、休息模式、双语切换及轻量设置等功能展开实现。系统基于 React 框架开发，并借助 LocalStorage 完成任务与设置数据的本地持久化。研究结果表明，本项目能够将传统任务管理工具的“记录事项”模式扩展为“支持行动启动与持续专注”的学习支持模式，在教育信息化与人机交互设计结合的层面具有较好的应用价值。

关键词：ADHD 友好设计；学习规划；时间管理；番茄工作法；Web 应用

引言

随着数字化学习环境的不断发展，学生在课程学习与项目实践中越来越依赖各类生产力工具辅助安排任务、分配时间与跟踪进度。然而，大量通用型待办与项目管理工具往往更强调任务记录、分类与提醒，而对“如何帮助用户立即开始一项任务”“如何降低注意力切换成本”“如何在专注后维持节奏”等关键问题缺乏针对性支持。已有研究指出，注意缺陷多动障碍及相关执行功能障碍与计划能力、组织能力、工作记忆及任务完成效率密切相关[1][2]。同时，大学阶段学生的执行功能表现与学业成绩之间也呈现明显联系[3]。因此，围绕学习任务启动困难与持续专注困难，开发一个更符合特定用户需求的 Web 应用，既具有现实意义，也具备一定的研究与实践价值。基于此，本文围绕 FocusFlow Study Planner 的需求背景、系统设计、技术实现与应用价值进行总结，以说明其作为课程项目应用实现部分的合理性与可行性。

# 1 项目背景与需求分析

## 1.1 现实问题分析

在实际学习过程中，学生面临的困难并不只是“任务太多”，更深层的问题在于面对复杂任务时往往难以迅速进入执行状态。例如，用户可能知道需要完成“论文初稿”“课程展示”或“系统开发”，但由于缺乏明确的切入点，任务会长期停留在抽象层面，最终形成拖延。与此同时，当应用界面一次性展示大量事项、标签、分类和说明时，用户还会在“查看任务”这一阶段就消耗过多认知资源，从而进一步削弱行动意愿。对注意力调节能力较弱的用户而言，这种问题尤为突出。

## 1.2 目标用户定位

本项目的目标用户主要是高校学生与研究生，尤其是存在以下特征的用户群体：第一，容易被复杂任务压倒，不知道从哪里开始；第二，面对过长的任务列表会产生明显的视觉压力；第三，需要借助外部结构帮助自己保持专注节奏；第四，希望使用简单、直接且支持中英文切换的学习工具。需要指出的是，虽然本项目强调 ADHD 友好设计，但其设计结果并不局限于 ADHD 用户，也适用于任何偏好低噪音、低负担界面的普通学生用户。

## 1.3 项目目标

本项目的总体目标是构建一款能够支持学生快速启动任务、进入专注状态并保持节奏的 Web 应用。具体而言，包括以下三个方面：一是通过更低门槛的录入方式，让用户可以迅速捕捉任务，而不是在填写复杂表单时失去耐心；二是通过今日聚焦与最小下一步设计，将抽象任务转化为可立即执行的动作；三是通过全屏倒计时与休息模式构建完整的专注闭环，使应用不再停留于“写下任务”，而是延伸到“真正开始执行任务”。

# 2 系统设计与功能实现

## 2.1 任务录入与最小下一步设计

FocusFlow 采用低阻力录入策略，用户在创建任务时仅需输入任务标题、课程或项目名称、截止日期、精力级别以及“最小下一步”。其中，“最小下一步”是本项目区别于一般待办工具的重要设计点。与其要求用户面对抽象任务，不如提示用户将任务拆解为一个具体且立即可执行的动作，例如“打开文档并写三句”“阅读一篇文献并标注两段重点”。这种处理方式有助于缩短从计划到行动之间的心理距离，从而提升任务启动概率。

## 2.2 今日聚焦与任务筛选

在任务展示方面，系统并未简单地把所有任务都平铺到首页，而是加入了“今日聚焦”模块，仅保留少量最值得立即开始的事项。这样做的目的在于减少同时出现在视野中的任务数量，降低用户面对界面时的压力感。除此之外，系统还保留任务状态筛选、课程筛选和精力级别筛选等管理功能，用于满足用户对整体任务库进行浏览和控制的需求。由此可见，项目在“低认知负担”和“基本管理能力”之间进行了平衡。

## 2.3 番茄钟式专注模式

当用户点击“现在开始”后，页面会进入全屏专注模式。此时界面主体转变为一个大号倒计时计时器，当前任务名称、下一步动作及状态信息被集中呈现，而其他界面元素则显著弱化。这种处理方式吸收了番茄工作法关于时间切片和单任务专注的优点[4]，使系统从单纯的规划工具扩展为执行辅助工具。相比于仅在任务旁边显示状态变化，专注模式通过全屏沉浸式界面强化了“现在只做这一件事”的交互意图，更适合任务启动困难用户的使用场景。

## 2.4 休息模式与轻量设置

为了避免用户在一轮专注结束后再次失去节奏，系统进一步引入了休息模式，并支持在专注结束后自动进入休息轮次。用户还可以在设置面板中调整启动时长、标准专注时长、深度专注时长、休息时长以及是否自动进入休息。这样的设计使工具在保持简洁的同时具备基础个性化能力，从而更贴近真实学习环境下不同用户的节奏偏好。

## 2.5 双语界面与视觉表达

本项目支持中文与英文界面切换，并通过昼夜主题切换进一步增强使用灵活性。双语设计一方面能够满足用户日常使用需求，另一方面也便于项目在课程答辩与成果展示中进行中英文环境下的演示。视觉表达上，系统采用较柔和的背景、控制文字数量并减少说明性段落，使页面更像可直接使用的工具，而非用于展示理念的宣传页。

# 3 技术实现与项目特点分析

## 3.1 技术实现

从实现层面看，本项目基于 React 前端框架构建，采用函数式组件与 Hooks 进行页面状态管理。任务数据、语言偏好及专注设置均通过浏览器 LocalStorage 完成本地持久化，以保证系统具备基本可用性与离线状态下的连续性。系统内部实现了任务创建、编辑、删除、筛选、排序、聚焦展示、倒计时控制、休息轮次切换与设置保存等逻辑。同时，项目还通过自动化测试验证了关键交互流程，包括双语切换与进入专注模式等核心行为。

## 3.2 项目特点

FocusFlow 的主要特点可归纳为以下几点。第一，它不是围绕“功能越多越好”展开设计，而是围绕“是否更容易开始行动”进行取舍；第二，它在产品逻辑上将任务拆解、任务聚焦、专注计时与休息管理连接为一条连续路径；第三，它通过减少说明性文案、弱化无关信息的方式，尽量降低用户在界面上的认知消耗；第四，它同时具备一定展示性与可运行性，既适合课程项目交付，也能直接作为在线工具原型进行演示。

## 3.3 与一般待办工具的差异

一般待办工具更关注事项罗列、项目归类和完成打勾，而 FocusFlow 更关注任务启动支持与专注执行支持。前者强调“管理任务”，后者强调“帮助行动”。其差异主要体现为：一是通过最小下一步降低任务启动难度；二是通过今日聚焦减少任务暴露数量；三是通过全屏倒计时让用户真正从规划进入执行；四是通过休息模式形成更完整的行为闭环。因此，本项目虽然以任务管理为表面形式，但本质上更接近一个以执行功能支持为导向的学习辅助系统。

# 4 项目价值与课程适配性分析

## 4.1 项目应用价值

从应用价值看，FocusFlow 能够帮助学生在复杂学习任务中更快进入状态，并通过结构化时间管理降低拖延和混乱感。对于有注意力调节困难的用户而言，这种价值尤为突出。项目将“任务可视化”“行动提示”“时间切片”与“休息恢复”结合起来，使时间管理工具从静态记录方式转向动态行为支持方式。

## 4.2 与课程要求的匹配

本课程强调的是围绕现实问题开发移动或 Web 应用，并在应用设计中体现信息技术方法与开发能力。FocusFlow 具备明确的问题背景、清晰的目标用户、完整的 Web 应用实现、可解释的设计原则以及可扩展的技术方向，能够较好地满足课程项目对“现实问题导向”和“应用实现能力展示”的双重要求。项目虽然当前仍属于前端原型性质，但它已具备较清晰的产品定位与系统功能结构，能够支撑课程论文与应用演示两部分的成果呈现。

# 5 结语

本文围绕 FocusFlow Study Planner 的项目背景、需求分析、系统设计、技术实现与应用价值进行了总结。项目以 ADHD 友好设计为出发点，通过低阻力任务录入、最小下一步、今日聚焦、番茄钟式全屏倒计时和休息模式等功能，尝试解决学生在学习场景中任务难以启动、界面容易过载、专注难以持续等现实问题。综合来看，该项目不仅具备实际应用意义，也能够在课程作业中较好地展示 Web 应用开发、界面交互设计与用户需求分析的综合能力。未来若进一步引入账户系统、云端同步、学习数据统计与个性化推荐等模块，系统仍有较大的扩展空间。
